LeetCode Problem: Number of Submatrices That Sum to Target

Problem Description

The problem "Number of Submatrices That Sum to Target" presents a scenario where we are given a matrix and a target value. The task is to determine the number of non-empty submatrices in the matrix whose sum is equal to the target.

A submatrix is defined by specifying the top-left and bottom-right coordinates, (x1, y1) and (x2, y2) respectively. A submatrix contains all the cells within the specified coordinates.

Here's an example to illustrate the problem requirements:

Example:

Input: matrix = [[0, 1, 0], [1, 1, 1], [0, 1, 0]], target = 0

Output: 4

Explanation: There are four 1x1 submatrices that only contain 0.

Solution

To solve this problem, we can use a combination of matrix manipulation and hashmap. The solution can be implemented with the following steps:

  1. Create a new matrix A of the same size as the input matrix matrix.
  2. Iterate over each row of the matrix and calculate the cumulative sum for each cell.
  3. Store the cumulative sums in the matrix A, where A[i][j] represents the sum of all cells from (0, 0) to (i, j).
  4. Initialize a hashmap hm to store the frequency of sum values.
  5. Iterate over all possible submatrices by considering all combinations of the top-left and bottom-right coordinates.
  6. For each submatrix, calculate its sum by subtracting the cumulative sums of the cells outside the submatrix.
  7. Check if the difference between the submatrix sum and the target exists in the hashmap hm.
  8. If the difference exists, increment the counter by the frequency of the difference value.
  9. Update the hashmap with the current submatrix sum frequency.
  10. Return the final counter as the number of non-empty submatrices that sum to the target.

Based on the problem requirements and your provided solution, we can implement the following solution:


class Solution {
    public int numSubmatrixSumTarget(int[][] matrix, int target) {
        int h = matrix.length;
        int w = matrix[0].length;
        int A[][] = new int[h][w];
        int sum;
        int counter = 0;

        // Calculate cumulative sums for each cell
        for (int i = 0; i < h; i++) {
            sum = 0;
            for (int j = 0; j < w; j++) {
                sum += matrix[i][j];
                A[i][j] = sum;
                if (i > 0) {
                    A[i][j] += A[i - 1][j];
                }
            }
        }

        // Use hashmap to store sum frequencies
        HashMap hm = new HashMap<>();

        // Iterate over all possible submatrices
        for (int i = 0; i < w; i++) {
            for (int j = i; j < w; j++) {
                hm.put(0, 1);
                for (int k = 0; k < h; k++) {
                    sum = A[k][j];
                    if (i > 0) {
                        sum -= A[k][i - 1];
                    }
                    if (hm.get(sum - target) != null) {
                        counter += hm.get(sum - target);
                    }
                    hm.put(sum, hm.getOrDefault(sum, 0) + 1);
                }
                hm.clear();
            }
        }
        return counter;
    }
}

Complexity Analysis

The time complexity of this solution is O(w^2 * h), where w and h are the width and height of the input matrix, respectively. This is because we iterate over all possible combinations of the top-left and bottom-right coordinates, which takes O(w^2) iterations, and for each iteration, we calculate the sum of the submatrix, which takes O(h) operations. The space complexity is O(w * h) to store the cumulative sums in the matrix A and the hashmap hm.
